# 通知渠道总览

网页侦探支持多种通知渠道，可任意组合，单任务可同时绑定多个渠道。

## 支持的渠道

| 渠道 | 是否需要外部配置 | 适合场景 |
| --- | --- | --- |
| 📱 pushplus | ✅ 需要 pushplus token | 推送到个人微信 |
| 💻 本地桌面通知 | ❌ | 个人使用，Windows 弹窗 |
| 🟡 钉钉机器人 | ✅ 群机器人 Webhook | 团队群通知 |
| 🟢 企业微信机器人 | ✅ 群机器人 Webhook | 企业内部群 |
| 🔵 飞书机器人 | ✅ 群机器人 Webhook | 飞书群通知 |
| 🛰 自定义 Webhook | ✅ 你自己的 HTTP 端点 | 接入自建系统 |
| ✉ 邮件 | ⏳ 高级会员开放 | 重要变更归档 |

## 在哪里管理通知渠道？

「我的 → 通知渠道」是统一管理中心：

- 添加新渠道
- 编辑现有渠道
- **测试** 渠道是否能正常发送
- 删除渠道

::: tip 配置一次，多处复用
通知渠道在「我的」里统一配置，多个任务可以选用同一个渠道，
不需要每个任务单独配置一遍。
:::

## 添加通知渠道

1. 进入「我的 → 通知渠道 → 添加渠道」
2. 选择 **渠道类型**
3. 输入 **渠道名称**（自定义，便于辨识）
4. 填写各渠道所需的配置（如 token、webhook url）
5. 点击 <kbd>测试</kbd> 验证 → 成功后保存

各渠道的具体配置：

- [pushplus](./notify-pushplus)
- [本地桌面通知](./notify-local)
- [钉钉 / 企微 / 飞书](./notify-im)
- [自定义 Webhook](./notify-webhook)

## 任务绑定渠道

创建 / 编辑任务时，「通知渠道」一栏可以勾选 **任意多个** 已配置的渠道。

::: tip 多渠道冗余
建议每个任务至少绑定 **本地通知 + 1 个远程渠道**，
互为兜底，避免漏接消息。
:::

## 消息内容格式

不同渠道的消息内容略有不同，但都会包含：

- 任务名称
- 触发时间
- 网页 URL
- 变化前 / 变化后的内容片段
- 直达链接（点击跳到目标页面）

## 失败重试策略

- 单次发送失败：最多重试 **2 次**，间隔 **5 秒**
- 重试仍失败：写入消息记录的「失败」状态
- 全部远程渠道失败：自动写入本地通知作为兜底（如果绑定了本地渠道）

## 测试渠道

每个渠道在「我的 → 通知渠道」列表里都有 <kbd>测试</kbd> 按钮，
点击会立即发送一条测试消息，用于验证配置是否正确。

测试消息示例：

```
【网页侦探 · 测试】
这是一条来自网页侦探的测试通知。
如果你收到了，说明该渠道已配置成功 ✅
```

## 删除渠道

- 「我的 → 通知渠道」列表里点 <kbd>删除</kbd>
- 删除前请确认 **没有任务还在使用该渠道**
- 删除后，绑定它的任务会继续运行，但不再向该渠道发送通知
