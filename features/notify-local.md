# 本地桌面通知

本地桌面通知是 **零配置** 的内置渠道，会在 Windows 右下角弹出系统通知。

## 配置方式

进入「我的 → 通知渠道 → 添加渠道」，类型选择 **本地通知**：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 渠道名称 | ✅ | 自定义，如「桌面提醒」 |
| 启用声音 | - | 默认开启，弹窗时播放提示音 |
| 重要程度 | - | 普通 / 重要（重要时弹窗不会自动消失） |

不需要任何外部 token。

## 显示效果

- Windows 10/11 系统通知样式
- 包含任务名称、变化摘要、点击跳转链接
- 点击通知可跳转到任务的「执行记录」详情

## 不显示通知？

如果配置后收不到本地通知，请按顺序检查：

### 1. Windows 通知是否被禁用

`设置 → 系统 → 通知和操作`：

- 「获取来自应用和其他发送者的通知」必须为 **开启**
- 找到「网页侦探」，确认它的通知也是开启状态

### 2. 是否处于「专注模式」

`设置 → 系统 → 专注助手`：

- 处于「仅优先级」「仅闹钟」时，通知会被静默
- 可以把「网页侦探」加入优先级列表

### 3. 客户端是否在运行

本地通知依赖客户端进程，关闭后无法弹出，请确保 **客户端最小化到托盘** 而不是退出。

### 4. 多显示器下通知去哪了？

Windows 默认通知在 **主显示器右下角** 弹出，
如果你有多个显示器，确认主显示器设置。

## 与远程渠道的关系

本地通知是 **兜底机制**：

- 任务可同时绑定本地 + 远程渠道
- 远程渠道失败时，会自动 fallback 到本地通知
- 即便没有网络也能发出本地提醒

## 不打扰模式

每个本地通知渠道都可以配置 **静默时段**，例如：

- 22:00 - 08:00 内不弹窗
- 仅写入消息记录，等次日早晨再统一回看

## 隐私

- 本地通知 **不经过任何外部服务器**
- 通知内容仅在你本机系统内传递
- 关机或重启后未读通知不会保留（Windows 系统级行为）
